HTML

웹사이트를 기획하면 사람이 하는 일과 기계가 하는 일로 분리할 수 있다.
사람이 하는 일: 코드(code), 소스(source), 언어(language) 등 프로그래밍 언어라는 표현을 쓴다.
기계가 하는 일: 애플리케이션(application), 응용 프로그램[앱(app), 프로그램(program)], 웹페이지(web page) 모이면 웹사이트(web site) 라는 표현을 쓴다.

실습환경

Atom이라는 에디터를 사용

기본 문법과 태그

HTML에는 태그(tag)라는 문법이 존재 <strong> ~ </strong> : 강조 태그 <u> ~ </u> : 밑줄 태그 <h1> ~ </h1> : h1 부터 h6 까지의 태그가 있다. 제목 태그 (숫자가 커질수록 글씨가 작아짐.) <br> : 단순 줄 바꿈 <p> ~ </p> : 단락 <img src="원하는 이미지 주소" width="100%"> : 이미지 표시, 이미지 크기 조절 <li> ~ </li> : 목차 or 목록 <ul> ~ </ul> : <li> 태그의 부모 태그 li 태그를 묶어 항목들을 구분 가능 <ol> ~ </ol> : 항목 앞에 자동으로 숫자가 매겨짐

문서의 구조와 슈퍼스타들

<title> ~ </title> : 페이지의 제목 <meta charset="utf-8"> : utf-8 문서를 읽음, 글이 깨지지 않음 <head> ~ </head> : 본문 설명 <body> ~ </body> : 본문 <html> ~ </html> : 최고위층 태그 <!DOCTYPE HTML> : <html> 태그 위에 작성하며 관용적으로 이 문서에는 HTML이 담겨 있다는 뜻

HTML의 제왕

<a href="링크를 걸 주소" target="_blank" title="~"> ~ </a> : 주소 링크, 새 탭에서 열림, 툴팁 나타남

기본적인 태그들과 html 문법들을 배워보았다.
